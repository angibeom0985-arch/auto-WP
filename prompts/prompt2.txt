'본문 지침'
- 각 본문은 600자 이상으로 구성하고, <p>코드를 사용해.
- 본문의 중요한 내용은 <strong> 또는 <u> 코드 사용해.
- <h3> <ol> <ol> 코드를 적절히 사용하고, 리스트 코드 쓸 때, 맨앞에 숫자나 기호 사용해서 더욱 가독성을 높여. 
- 문장이 너무 길어질 경우, </p>로 닫고 새롭게 <p>를 사용해. 적절한 공백을 활용함으로써 가독성을 높여.
- '클릭'이란 단어 대신에, 유의어 사용해. 본문의 문장은 '~해요'체로 작성하며, 전문 용어 쓸 때는 초등학생도 이해할 수 있도록 쉽게 풀어서 설명할 것.
- 개인적 또는 대중적인 경험 기반의 서술할 것. (직접 겪은 상황, 주변 사례, 후기 등) 실용적인 해결법, 단계별 팁, 주의사항, 예시 등을 포함하여 독자의 상황을 구체적으로 공감하고 안내할 것. 
- 단순 정보 나열이 아닌, 실제 써봤던 과정, 실수했던 경험, 좋았던 팁 등의 콘텐츠를 강조하여 Google의 E-E-A-T 평가를 만족시킬 수 있도록 신뢰할 수 있는 데이터를 인용할 것. 
- 본문 내용에 '신청', '조회', '다운로드', '홈페이지' 등의 단어가 포함되어 외부 링크가 필요할 경우, <a href="{url}" target="_self">앵커 텍스트</a> 형식으로 링크를 첨부할 것. 이때 '앵커 텍스트'는 '{keyword}'와 관련하여 독자의 행동을 자연스럽게 유도하는 문구로 직접 작성해. '클릭'이라는 단어는 절대 사용하지 마.
- <html>, <head>, <title>, <body>, <article> <br> 태그는 절대 사용하지 않을 것.
